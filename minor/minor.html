Desktop

Field

Lab

Workshop

What have I learned?

  • Setting up the React-Native DEV environment
  • Using state hooks to update the state of components
  • Importing RN components and adding dependencies to 'package.json'

Motivation

For the 'Hybrid App' assignment I needed to pick the most suitable hybrid framework for the concept. Prior to this POC I already had experience with React-Native. However, this was a while back so it was necessary to see if I still remembered the basics. This POC helped me refresh my memory and helped me make my choice on what framework to choose.

Click counter

A simple React-Native click counter

1

First I installed React-Native with help of the official documentation. There are a lot of requirements in order to use React-Native. Node.js, a JDK and Android Studio with certain SDK's have to be installed.

2

I followed the 'learn the basics' documentation of React-Native which first gives an example of importing components and making a View with a simple Text component with 'Hello World' as the string.

import React from 'react';
import { Text, View } from 'react-native';

const HelloWorldApp = () => {
  return (
    <View
      style={{
        flex: 1,
        justifyContent: "center",
        alignItems: "center"
        }}>
        <Text>Hello, world!</Text>
    </View>
  )
}
export default HelloWorldApp;

3

The documentation explained about using state in React-Native which allows components to have their output changed. A click counter is an example where state needs to be applied since the output has to be changed.

return (
    <View style={styles.container}>
        <Text>You clicked {count} times</Text>
            <Button
                onPress={() => setCount(count + 1)}
                title="Click POC"/>
    </View> 
);

Result


Unfortunately the screen recording of the emulator wasn't working well.

Evaluation

What I like about React-Native is that the styling is very similar to CSS. This makes it easy for me to know what properties I need. However, from my prior experience with React-Native I remembered that I struggled to apply 'state' and 'props'. This is vital to understand since it can make an app dynamic. In Technical Research I discussed the advantages and disadvantages I found further.

Concept

Initial concept

At the start of the hybrid app project a few concepts were already defined by the teachers and put on a Powerpoint presentation. I was interested in the following concept:

Visitors to a pop concert can (if allowed by the organisation of the concert) stream live video (and music) to a server with this app. Users can then log on to that server and follow the concert live on screen.

I was interested in this concept, because I had never done something that involved livestreaming technology before. I also saw that this concept offered a complicated design challenge since there's three different kind of stakeholders involved (visitors at concert, viewers at home, organizers of concerts). As ICT & Media student this convinced me to take up this challenge.

Final concept

After the market research was done, I redefined the concept since the conclusion of the market research was that there is a niche in the market for livestreaming apps that focus on festivals instead of specific artists or concerts.

The final concept:

Visitors to a pop festival can (if allowed by the organiser of the festival) stream live video (and music) to a server with this app. Users can then log on to that server and follow the concerts of the festival live on screen.
