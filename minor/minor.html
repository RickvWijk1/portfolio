Brainstorming

I started looking for ways to get the ideation process going. I suggested the 'Lotus Blossum' method which is recommended by the Digital Society School.The idea is that you start with a central theme and from that theme, eight conceptual themes flow out of the central theme. Each of them are used as central theme to generate 8 more themes and so on. You use this when there is a need to generate a big number of ideas quickly. It's similar to mindmapping or making word webs. I used this method during my internship with success.

We ended up not using this method, but simply started making a word web with Zenya as the central theme. After we wrote some words down it felt like making a word web didn't give us any inspiration/ideas.

Design Sprint method

Ruben found another method that's used to help ideating. It's usually used across a whole week and this process gets repeated multiple times during a project. In our case, we cut it short to an average of two days each time this method was used.

This is how we used it:

1

Write down the main question. What do you want to achieve with this project?

2

Each member writes down as many HMW/HCW questions on sticky notes as they can. These questions help break down the main question.

3

Each member gets a couple of votes to give to specific HMW/HCW (How Might We, How Can We) questions that they think are the most important.

4

Define the routes/steps the user has to take until the user reaches their goal. This is a form of User Journey Mapping.

5

Order the HMW/HCW questions under the user routes/steps where the project team thinks they belong. The questions with the least votes get stored away and used if they're important in future sprints.

6

Come up with arguments on why this project can't succeed. Translate them into research questions.

7

Some of the questions need to be answered to find a possible solution. Each member creates a storyboard that holds a possible solution to the user's problem.

User interaction

To make it clear (for myself and others) how the user interaction will work, I made a user scenario.

Prev Next

GUI AR glasses

The users need a way to navigate to specific information and a way to understand/consume the information efficiently. My plan was to first start of with basic actions like navigating. The user should have multiple options to achieve something. For example, a surgeon might need to interact with the protocols handsfree and for this reason can't use the buttons or touchpad. Voice control is the better option in this case.

The menu is on the left. With this menu users can navigate between screens that have different functionality.

Users can choose to toggle the camera, because in some cases users might only need to focus on the UI (in fullscreen) and not need to see their surroundings. For example, when a mechanic needs to get information about a specific car part reparation they might only want to read some instructions and then perform these actions.

This screen shows information on how to use Zenya. Since very little people have experience with AR glasses they'll have to figure out what functions the glasses have and how to use them. There's many ways for the user to interact with the app, e.g. swiping, tapping, clicking, talking.

This screen is where users can search for/find protocols. They can scroll through the available protocols by swiping or they can try to find a specific protocol by using voice control.

This screen is where users can adjust settings to their liking/situation. I didn't finish designing this since this wasn't part of the core concept.

This is the design for the so-called 'stationary protocols'. These are the protocols that are fixed to one place. For example, a disinfection sink can be recognized by the glasses so that it will display this screen that will let the user know that they can start the protocol.

This screen contains the first instruction the user gets when a protocol is started.

Employees/users can make comments on the protocol's instructions. This way they can help improve/help keep the protocol up to date.

Prev Next

During and after the design iteration I started to see some (potential) threats/problems:

  • Size of elements (text, icons, images, etc.), are they not too small/big? Can the user clearly see and understand them?
  • Contrast, is the trademark/primary (pink) Zenya colour in combination with white text convenient to read? What if the user finds themselves in a very bright room facing lights? Will they be able to read the text?
  • Blocking vision, can the user still look around them when the UI is cluttered?

    • The only way to know if these are problems is by testing the design

Structs

At first when I was working on the Scrumdinger app, structs seemed very similar to classes to me. I then did some research and found out that structs are value types and classes are reference types. Classes have more functionality, they can do anything structs can and more. Classes can inherit from other classes and have deinitialisers. However, using structs is better practice if they're relatively small and copiable, because copying is safer than having multiple references to the same instance which is what happens with classes.

Protocols

Before the Scrumdinger workshop I'd never heard of protocols before. I was wondering if it wasn't some kind of synonym for methods. When I did some research I found out that it's used to define a sort of blueprint of methods, properties and other requirements that suit a particular task or piece of functionality.

I used the 'Identifiable' protocol in the Scrumdinger app to create a unique id for each instance.

struct DailyScrum: Identifiable {

Vision framework

I found out that the Vision framework can be used to track gestures. I downloaded this sample project that Apple provides. It showcases how you can track the top of your thumb and index finger. Anchor points are added to these spots and if the anchor points overlap eachother it triggers an event that allows you to draw. You can use the 'VNDetectHumanBodyPoseRequest' to detect key body points. It already provides resources to track common body parts (right ankle, left knee, right ear, etc.). 

// Get the CGImage on which to perform requests.

// Create a new request to recognize a human body pose.
let request = VNDetectHumanBodyPoseRequest(completionHandler: bodyPoseHandler)

do {
    // Perform the body pose-detection request.
    try requestHandler.perform([request])

Vision of the future

What have I learned?

  • Learned the (possible) dangers of AR and VR
  • Learned in what ways AR and VR could help people in their every day activities.

Motivation

I want to take a moment to talk about what I expect the future of mobile technology to look like and what impact this technology will have in our daily lives. I believe it's important to talk about the rules (laws and ethics) that need to be set for new technologies, because if boundaries aren't set, organisations will start to abuse these technologies for their own gain. Some organisations/individuals might not even realise they went too far, because no rules are set.

It is also important for me on a personal level, because I will soon be working for organisations that are active in the IT industry and I will, mostly on an ethical level, be responsible for the usage of technology.

Across the semester I've participated in workshops that made me think about the technologies of the future. AR (Augmented Reality) was the most discussed and practiced subject during the semester. For this reason I will talk about what I think of AR in its current state and about what I expect from AR in the future.

AR - the current state

Right now AR is still a work in progress. Unlike our smarthpones it isn't part of our daily lives yet. Yet AR is applied/used in some areas already:

  • Surgeons and other professions use AR glasses to allow unexperienced employees to watch along with experienced employees remotely.
  • Entertainment: Snapchat has a wide variety of interactive filters people can use to entertain themselves. Snapchat also made interactive AR games (called Snappables). They track your touch, motion and facial expressions to compete for high scores or in literal head-to-head multiplayer match-ups

At the moment most organizations are still figuring out where and how they can apply AR to improve their products/services.

AR - the future

I expect AR to improve experiences and daily activities of people in a flexible way to the point where AR will become part of our daily routines just like our smartphones are now. I expect this, because AR already shows that it is capable of it. Take a look at the Group Project for example. I can take the Vuzix AR glasses and get information handsfree which is more flexible than needing to take your phone out and keep holding it. Have you ever had to screw a cabinet together? I did. I had to drop my tools constantly, because I had to go through the manual.


In the near future more and more people will wear these glasses and a lot further in the future I expect to see a lot of other wearables/devices that support the usage of AR. For example, I believe we will eventually see some sort of smart contact lenses that have the ability to augment your vision. I also expect support for interaction with AR. Perhaps some sort of advanced smart rings that can track all movement of your fingers, hands or even your arms. This can make air gestures very accurate and thus improving the interaction meaning touch screens will become less relevant.

In the future humanity and technology will 'melt' together more and more
Here's some of the advantages that AR might bring us:
  • Socially: AR allows us to get information quicker than before, which also means it allows us to share it faster with others. This can be useful in a work environment.
  • Immersiveness: AR will be able to render very realistic digital content into our environment. This provides opportunities for the entertainment sector. Especially in combination with wearables that support surround sound (which some AR glasses already support)
I can also see some serious problems arise:
  • Privacy: if we'll have AR devices (glasses or lenses) on us the whole day, a lot of information can be tracked. If the technology is not secure, our POV/sight will be exposed. Everything we see can be tracked in combination with our location.
  • Propaganda: if we'll have AR devices (glasses or lenses) on, organisations can show us AR content that is so realistic/matches so well with the real world that we can't tell what's real and what's fake. Organisations like political parties could use this to their advantage; they could force a narrative on us. They could stage whole scenes and we wouldn't even know that it is fake.
  • Mentally: You're connected to the internet longer if you are wearing AR glasses/lenses. Too much information (24/7) might cause mental exhaustion. Especially when there's social pressure involved (e.g. you need to answer everyone's messages) it could exhaust you.
    • What if our lives become staged?

I don't think we should be afraid of civilisation ending or AR completely corrupting us. Humanity has been innovating forever and so far it hasn't doomed us. Sure, there will be struggles. I expect there to be sensitive data leaks just like we have now (Facebook for example). As long as we keep setting rules, in the form of laws and ethics, I don't think AR technology will get out of hand.

I think we should prepare ourselves for future technologies like AR by keep adding/maintaining the laws and raising ethical awareness.

My plans

I'm mostly interested in interaction design and frontend development. I think that in the future a lot of apps will require the applying of interaction design. This is also why I expect that the job title of 'frontend developer' and '(interaction) designer' will become more similar/overlap each other or require an even closer collaboration than they do now. I will search for an internship that requires interaction design and frontend development. When I've graduated and enjoyed the internship I'll look for a job that is similar.
